# خطة تنفيذ نظام الدفع العالمي الجديد (MoneyMotion)

سأقوم بتنفيذ النظام المتكامل الذي طلبته والذي يربط بين واجهة المستخدم، بوابة الدفع، وقاعدة البيانات بدقة عالية.

## 1. تحديث خدمة الدفع (Frontend Service)
*   **الملف:** `src/lib/moneyMotionService.ts`
*   **الإجراء:** تحديث كود إنشاء جلسة الدفع ليستخدم المفتاح الجديد `mk_live_CCupGS0FlIzOB9G6wbxA6w4GpFKzlAti`.
*   **الهدف:** ضمان إرسال `intent_id` و `email` و `product_id` داخل حقل `metadata` بدقة لضمان المطابقة 100%.

## 2. تحسين الويب هوك (Edge Function)
*   **الملف:** `supabase/functions/moneymotion-webhook/index.ts`
*   **الإجراء:** تعديل منطق استخراج البيانات ليتطابق تماماً مع هيكلية Payload التي زودتني بها (خاصة `paymentMethodInfo`).
*   **المميزات:**
    *   التحقق من التوقيع `HMAC-SHA512` باستخدام السر `32fb9...`.
    *   مطابقة ذكية (Smart Matching) باستخدام `intent_id` ثم البريد الإلكتروني كبديل.
    *   إرسال الفاتورة عبر Brevo مع التمييز البصري (Sinki vs Cheatloop).

## 3. تحديث لوحة الأدمن (Admin Dashboard)
*   **الملف:** `src/components/admin/AdminDashboard.tsx`
*   **الإجراء:** إضافة قسمين جديدين في القائمة الجانبية:
    1.  **Global Pending (طلبات الانتظار العالمية):** يعرض الطلبات التي بدأت الدفع ولم تكتمل.
    2.  **Global Completed (الطلبات المكتملة العالمية):** يعرض المدفوعات الناجحة مع تفاصيل البطاقة والمفتاح المصروف.

## 4. قاعدة البيانات (SQL)
*   سأزودك بكود SQL شامل لإنشاء:
    *   الجداول اللازمة (`webhook_logs` للتتبع).
    *   طرق العرض (`Views`) الخاصة بالأقسام المنفصلة.
    *   إعدادات النظام.

## 5. الاختبار التجريبي
*   سأوضح لك كيفية إجراء اختبار كامل للنظام للتأكد من تدفق العملية من الدفع إلى استلام الفاتورة.
